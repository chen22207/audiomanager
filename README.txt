情感语音数据管理与服务平台
需求分析
　　语音是人类交流的基础，是我们交换信息最自然、最有效、最便捷的手段之一。情感语言识别近年来收到各个行业的高度重视。探索情感语言数据对人们工作能力和生活状态的改善有着重要意义，具有非常重要的应用价值。所以本次软件工程课题就是采集不同行业人士在“喜悦”、“悲伤”、“烦躁”等情感状态下的语音情感数据，建立一个实用语音情感数据库用于服务于人们。
　　
各功能点论述：
　　一、
　　用户注册登录平台，完善自己信息，发布语音测试表单（选择题或判断题、打标签<自定义标签和选择行标签>），语音可以顺序呈现，也可以随机产生，选择测试群体。
　　任务接收者接受任务，完成测试（备注完成进度）。同一个声音可以让用户听两次或者多次，根据两个结果是否一致来判断用户是否认真听语音答题。
　　同一个声音可以在不同的任务中参与测试，但是每一次测试都要记录下来。
　　二、
　　平台需要和第三方科研机构共享，管理里面的上传和下载功能吸引第三方科研机构贡献出自己的情感语音。上传通过测试后可以开放下载，上传和下载都是有限的，只有贡献了才可以去下载。
　　三、
　　每一个声音有自己的语音声学特征，声学特征要通过自动化提取工具提取（如能量曲线、共振曲线等），要在后台建立一个特征提取的引擎来帮助完成特征提取的任务，提取出来以后要把特征放在网上可视化展示。
　　一个声音，用户可以编辑它的一般属性，看它的特征，还可以看它每一次的测试结果。
　　四、
　　界面做到简洁友好，标签要方便使用。
　　搜索功能：根据要求把具体的服务搜索出来。 项林平
